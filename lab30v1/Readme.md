# Звіт до лабораторної роботи 30: Юніт-тестування з xUnit

## Мета роботи
Отримати практичні навички написання юніт-тестів для власного коду за допомогою фреймворку xUnit, а також застосувати різні типи перевірок та параметризовані тести.

## Виконання роботи

**1. Створення бізнес-логіки**
Згідно з першим варіантом було створено основний проєкт та реалізовано клас калькулятора. Цей клас містить п'ять базових математичних операцій: додавання, віднімання, множення, ділення та піднесення до степеня. Логіка методу ділення включає перевірку знаменника для контролю ситуації з діленням на нуль.

**2. Налаштування тестового середовища**
Створено окремий тестовий проєкт із підключеним фреймворком xUnit. Додано посилання на основний проєкт для забезпечення доступу до класу калькулятора.

**3. Написання юніт-тестів**
Розроблено десять незалежних тестів. Для покриття різних наборів вхідних даних використано атрибут Theory разом із набором значень InlineData. Для перевірки специфічних умов та очікуваних винятків застосовано атрибут Fact.

**4. Тестовані сценарії**
Покрито успішні сценарії для кожної математичної операції. Додатково реалізовано тести для граничних випадків: 
1) робота з від'ємними числами; 
2) спроба ділення на нуль; 
3) математичні правила нульового та від'ємного степеня; 
4) додавання дуже великих чисел;
5) множення на нуль.

## Результати перевірки
Запуск команди тестування в консолі показав успішне проходження всіх розроблених перевірок. Помилок чи збоїв у логіці калькулятора не виявлено. Код відповідає вимогам завдання та критеріям чистої архітектури.

## Висновок
Під час виконання лабораторної роботи засвоєно базові принципи автоматизованого тестування коду. На практиці доведено, що використання параметризованих тестів значно зменшує дублювання коду при перевірці однотипної логіки. Написані тести гарантують коректну роботу математичних операцій при різних вхідних даних та створюють надійний захист від випадкових поломок у майбутньому. Мета лабораторної роботи повністю досягнута.
